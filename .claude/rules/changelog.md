# 課題対応後のドキュメント更新ルール

課題（バグ修正・機能追加・スキル追加など）の対応が完了したら、
コミット・リリース前に以下のドキュメントを更新すること。

## CHANGELOG.md（必須）

- `CHANGELOG.md` に新しいバージョンのセクションを追記する
- フォーマットは [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) に準拠し、日本語で記述する
- 変更の種類に応じて `### 追加`・`### 変更`・`### 修正`・`### 削除`・`### その他` を使い分ける
- CLI オプション・ファイル名・コマンドなどの技術用語はそのまま（日本語に訳さない）

## README.md（必要な場合のみ）

以下に該当する変更が含まれる場合は README.md も合わせて更新する:

- 新しい CLI オプション・コマンドの追加
- インストール・セットアップ手順の変更
- 前提条件（依存ツール・バージョン）の変更
- スキルの使い方や機能説明の変更
